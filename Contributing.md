# Contributing to Awesome Free Mobile Apps

First off, thank you for considering contributing to this project! This list aims to catalog open source mobile applications across all major platforms.

## How to Contribute

### Adding a New App

1. **Fork the repository**
2. **Add your app** to the appropriate platform and category in README.md
3. **Follow the format** exactly as shown below
4. **Submit a pull request**

### App Entry Format

```markdown
- [**App Name**](GitHub or Repository URL) - Brief description in 1-2 sentences. Key features include major functionality that differentiates this app.
```

**Example:**
```markdown
- [**Signal**](https://github.com/signalapp/Signal-iOS) - Free, world-wide, private messaging and phone calls app with end-to-end encryption. Features include disappearing messages, screen security, and relay calls to protect your IP address.
```

## Description Writing Tips

### Good Descriptions

‚úÖ **Signal** - "Free, world-wide, private messaging and phone calls app with end-to-end encryption. Features include disappearing messages, screen security, and relay calls."

‚úÖ **Telegram** - "Messaging app focused on speed and security with cloud-based storage and sync. Supports channels, groups, bots, and file sharing with no size limits."

### Poor Descriptions

‚ùå "An amazing app for chatting" - Too vague, no specific features
‚ùå "The best messenger app ever made with incredible features" - Marketing language, not descriptive
‚ùå "Telegram" - No description at all

## Review Process

1. All pull requests will be reviewed for quality and accuracy
2. Maintainers may suggest improvements to descriptions
3. Apps must meet all quality guidelines
4. Duplicate apps or incorrect categorization will be flagged

## Code of Conduct

### Our Pledge

We are committed to providing a welcoming and inspiring community for all. Please be respectful and professional in all interactions.

### Our Standards

**Positive behavior includes:**
- Using welcoming and inclusive language
- Being respectful of differing viewpoints
- Gracefully accepting constructive criticism
- Focusing on what is best for the community

**Unacceptable behavior includes:**
- Trolling, insulting comments, or personal attacks
- Public or private harassment
- Publishing others' private information without permission
- Other conduct which could reasonably be considered inappropriate

## Recognition

Contributors will be recognized in:
- The main README.md (Contributors section)
- Release notes for significant contributions
- Special thanks for bulk additions

## Questions?
